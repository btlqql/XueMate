# XueMate 聚焦交付口径：面向小学的全面学习助手

> 交付定位：XueMate 是面向小学生的全面学习助手，不是单科工具，也不是单纯编程产品。答辩时可以用“编程启蒙/算法动画”作为一个清晰 demo 场景，但产品定位必须保持“小学全科学伴”。

## 1. 一句话定位

**XueMate 是一个面向小学生的全面学习助手，覆盖资料问答、作业理解、学习辅导、动画讲解、长期记忆和网页资料辅助。它可以服务数学、语文、英语、科学、编程启蒙等小学学习场景。**

## 2. 为什么仍然需要一个聚焦 demo

产品定位是全面学习助手，但答辩不能讲得太散，所以演示时选一个代表性场景：

```txt
小学编程启蒙：学生不理解冒泡排序 / 循环 / 列表索引
```

这个场景能同时展示：

| 能力 | 展示方式 |
|---|---|
| RAG | 读取老师课件、作业要求、代码规范 |
| Memory | 记住学生薄弱点：循环、索引、变量、边界条件 |
| 动画 | 把冒泡排序过程可视化 |
| 代码辅导 | 指出代码问题，但不直接代写 |
| Computer Use | 自动查找适合小学生/初学者的教学网页 |
| 本地数据 | 保存资料、对话、学习画像、复习队列 |

也就是说：**项目定位全面，演示内容聚焦。**

## 3. 产品范围

XueMate 面向小学学习，主要覆盖：

1. **问学伴**
   - 学科问答
   - 基于资料回答
   - 动画解释

2. **我的资料**
   - 导入老师课件、作业要求、笔记
   - 按科目分资料夹
   - RAG 资料检索

3. **工具箱**
   - 作业任务解析
   - 代码/报告辅导
   - 复习总结

4. **小实验**
   - 快速查资料
   - 操作网页
   - 固定实时网页区域

## 4. Demo 主线：小学编程启蒙

### Step 1：导入课程资料

入口：`我的资料`

导入示例：

```txt
Python循环与列表.pdf
排序算法课堂笔记.md
冒泡排序作业要求.txt
代码提交规范.pdf
```

系统动作：

- 文本抽取
- 章节感知分块
- embedding 入库
- 存入 SQLite 本地知识库

对应模块：

- `document.ts`
- `rag.ts`
- `db.ts`

### Step 2：学生提问

入口：`问学伴`

学生问：

```txt
冒泡排序为什么要两层循环？我总是不懂 i 和 j 是怎么变的。
```

系统动作：

- Hybrid RAG 检索老师课件
- 找到“相邻比较”“一轮最大值就位”“内层循环范围缩小”等资料
- 带 `[资料1]` 引用回答

对应模块：

- `rag.ts`
- `llm.ts`
- `chatStore.ts`

### Step 3：结合长期记忆

系统可能记得：

```txt
weak_point: for 循环嵌套不熟
weak_point: 列表索引容易越界
misconception: 以为每一轮都要比较到最后一个元素
preference: 喜欢动画解释、一步一步讲
```

回答策略：

- 不直接给完整答案
- 解释外层循环和内层循环分别负责什么
- 强调 `len(arr) - 1 - i`
- 提醒 `j + 1` 的越界风险
- 用小数组演示

对应模块：

- `memory.ts`

### Step 4：动画讲解

前端通过组件渲染动画，不依赖 SVG：

```txt
[5, 3, 8, 1]
比较 -> 交换 -> 最大值就位 -> 下一轮范围缩小
```

对应模块：

- `AnimationRenderer.vue`
- `SortAnimator.vue`
- `StepAnimator.vue`

### Step 5：代码辅导

入口：`工具箱 -> 学习辅导`

学生粘贴有问题的代码：

```python
for i in range(len(arr)):
    for j in range(len(arr)):
        if arr[j] > arr[j+1]:
            arr[j], arr[j+1] = arr[j+1], arr[j]
```

系统指出：

```txt
内层循环范围过大
j+1 可能越界
应该思考 len(arr)-1-i
```

重点：**给提示，不直接代写。**

### Step 6：自动找学习资料

入口：`小实验 -> 操作网页`

输入：

```txt
帮我找一个适合小学生的 Python 冒泡排序动画讲解网页
```

系统动作：

- 打开右侧固定“实时网页”区域
- 多模态模型看截图
- DOM 候选辅助定位搜索框/链接
- 自动输入、点击、滚动

对应模块：

- `computerUse.ts`
- `web.ts`
- `vision.ts`
- `AgentView.vue`

### Step 7：形成复习闭环

系统抽取记忆：

```txt
weak_point: for 循环嵌套
weak_point: 列表索引边界
misconception: 内层循环范围不缩小
reviewQueue: 冒泡排序循环边界 优先级90
```

这体现 XueMate 的核心价值：**不是问完就结束，而是持续陪伴学习。**

## 5. 评审时的四个核心亮点

### 亮点 1：小学资料问答，不是泛泛聊天

Hybrid RAG：

- nDCG@K 提升 8.55%
- Precision@K 提升 7.14%
- 上下文输入压缩 82.14%

文档：`docs/RAG_METRICS.md`

### 亮点 2：长期学习画像

Memory Atom：

- Prompt chars 减少 37.29%
- 重复事实减少 100%
- 过期事实抑制 100%
- 复习队列覆盖 100%

文档：`docs/MEMORY_SYSTEM.md`

### 亮点 3：小学生友好的组件化动画

适合解释：

- 数学步骤
- 科学实验过程
- 英语记忆方法
- 编程算法流程

### 亮点 4：网页辅助能力

Computer Use 可以帮学生自动找资料，但限定在内置实时网页区域，适合作为“学习资料辅助”。

## 6. 正确交付口径

不要说：

```txt
我们做了一个编程作业系统。
```

也不要说：

```txt
我们什么都做。
```

要说：

```txt
XueMate 是面向小学生的全面学习助手。它覆盖资料问答、作业辅导、动画讲解、长期学习画像和网页资料辅助。答辩中我们用“编程启蒙中的冒泡排序”作为代表场景，展示这套能力如何形成完整学习闭环。
```

## 7. 3 分钟演示脚本

1. 打开 `我的资料`，展示资料库可以按科目放资料。
2. 进入 `问学伴`，提问：`冒泡排序为什么要两层循环？`
3. 展示回答里引用资料、提醒循环边界、步骤讲解。
4. 展示冒泡排序动画组件。
5. 打开 `工具箱 -> 学习辅导`，贴一段有越界问题的代码。
6. 展示系统只给错误定位和提示，不直接代写。
7. 打开 `小实验 -> 操作网页`，输入：`帮我找一个适合小学生的 Python 冒泡排序动画讲解网页`。
8. 展示右侧固定实时网页自动搜索。
9. 最后讲：同一套能力也可以迁移到数学、英语、科学等小学学习场景。
